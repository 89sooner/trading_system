# Build execution checklist

Use this after coding and before finalizing implementation output.

## Inputs

- Approved plan or explicit scoped goal
- `AGENTS.md` policies in scope
- Planned validation commands

## Minimal evidence set

1. Goal restated in one sentence.
2. In-scope files identified and modified only as needed.
3. Validation commands executed and reported honestly.
4. Any blocked checks explained with reason.
5. Remaining limitations listed explicitly.

## Scope prompts

- Did any unrelated files change?
- Did any change exceed approved scope?
- Was any speculative refactor introduced?

## Validation prompts

- Build/lint/test executed in expected order when available?
- Are pass/fail outcomes recorded per command?
- Is there enough evidence for handoff to verify mode?
