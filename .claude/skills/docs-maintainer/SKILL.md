# Docs maintainer

Keep repository documentation aligned with the codebase.

## Workflow

1. Identify which user-facing behavior changed.
2. Update the minimum set of docs and examples that explain it.
3. Prefer concrete commands, file paths, and short examples.
4. Call out any intentional gaps or follow-up docs that still need work.

## Documentation targets

- `README.md` for setup, quick start, and top-level layout
- `docs/` for architecture or operator detail
- `configs/` and `examples/` when configuration shape or defaults changed
- `AGENTS.md` when repository workflow rules changed

## Output structure

### Updated docs
### What was clarified
### Remaining gaps

See `references/doc-sync-checklist.md` before finishing.
